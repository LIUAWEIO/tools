# 时区换算｜鸽鸽工具网

**立即使用：** [https://gegegj.com/calc/timezone](https://gegegj.com/calc/timezone)

## 功能介绍

将用户指定的**同一瞬间**，在多个预设时区中并列展示**当地日期与时间**（24 小时制中文格式）。与时间戳页互补：本页侧重「各地墙上时间」，时间戳页侧重「epoch 与 ISO 互转」。

远程协作、跨境发布与客服支持中，常见说法包括：**「北京时间 纽约时间」「UTC 转北京时间」「跨国会议时间」「夏令时换算」** 等。行业实践强烈建议在会议邀请中**同时写明 UTC 与主要参会城市本地时间**，减少「差一天」或「少算一小时」的沟通事故。

## 支持的参数

| 参数 | 说明 |
|------|------|
| 时间来源 | **本地日期时间**（`YYYY-MM-DDTHH:mm`）或 **Unix 时间戳**（秒/毫秒，解析规则同时间戳页） |
| 快捷 | 「使用当前时间」填充 |

1. 打开 [时区换算](https://gegegj.com/calc/timezone)。
2. 选择「本地日期时间」或「Unix 时间戳」，填写后查看表格各列。
3. 夏令时切换日前后，对关键会议再人工复核一次。

## 常见问题

**和 Unix 时间戳页有什么区别？**  
时间戳页解决数字与字符串互转；本页解决**多时区并列对照**。

**夏令时准吗？**  
各列由浏览器 `Intl.DateTimeFormat` 结合系统时区数据库计算，一般与主流 OS 一致；**法律或历史上的特殊时区规则**仍建议权威来源复核。

**跨国会议怎么写邮件最清楚？**  
推荐写法示例：`UTC 06:00 / 北京 14:00 / 纽约 01:00（日期）`。

**为什么和某世界时钟网站差 1 小时？**  
可能对方使用不同数据源或是否计入夏令时；关键会议请以多方工具交叉验证。

## 数据来源与计算依据

不调用远程世界时钟 API；依赖用户设备的 **IANA 时区数据** 与 **Intl** 实现。
