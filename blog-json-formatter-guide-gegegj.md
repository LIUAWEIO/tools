## 为什么要可折叠编辑

大型 JSON 往往层级深、字段多。折叠视图能快速隐藏不关注的区块，显著提升阅读和排错效率。

调试接口响应时，最先关注的通常不是每个字段，而是对象层级、数组数量和关键状态字段。可折叠编辑能先看结构，再逐层展开细节，比只看纯文本更高效。

## 格式化与压缩的使用场景

调试接口时用格式化提升可读性；线上传输或嵌入配置时可用压缩输出减少体积。

## 错误提示怎么利用

当 JSON 语法出错时，工具会给出中文错误说明和位置提示。优先检查中英文标点、键名引号和括号配对，通常能快速修复。

最常见的错误包括：对象键名没有双引号、使用中文逗号或中文冒号、数组或对象末尾多了逗号、字符串里混入未转义的换行。修复时建议从提示位置向前后各看一两行。

如果内容来自 JavaScript 对象、JSON5 或配置片段，可能包含注释、单引号、尾随逗号等标准 JSON 不允许的写法。需要先转换成严格 JSON，再进行格式化或校验。

## 格式化前的隐私和大文件注意事项

使用鸽鸽工具网的 JSON 格式化工具时，解析和压缩都在浏览器本地完成，不会把内容作为业务数据上传。即便如此，敏感生产数据仍建议先脱敏，再粘贴到任何在线工具中处理。

特别大的 JSON 可能让浏览器卡顿，尤其是包含大量数组项或深层嵌套对象时。遇到超大日志或导出文件，可以优先截取相关片段，或在本地使用 jq、Node.js 等命令行工具处理。
