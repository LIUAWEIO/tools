# 店里贴的收款码是静态还是动态？换海报还能扫吗？

**在网站上阅读：** [https://gegegj.com/blog/qrcode-dynamic-merchant-code-vs-static](https://gegegj.com/blog/qrcode-dynamic-merchant-code-vs-static)

**相关工具：** [打开工具页](https://gegegj.com/tool/qrcode)

## 静态码：一张图用很久

微信、支付宝个人/商户收款二维码多为静态：图案固定，顾客扫后手动输金额或由店主口述金额。打印贴玻璃、立牌即可，**换海报只要图案不变仍可调出同一账户**（账户别换）。

风险：静态码被他人覆盖换贴、截图传播，钱可能进错账；营业结束应收起或遮挡。

自己在 [二维码生成器](https://gegegj.com/tool/qrcode) 做的是普通 URL/文本码，**不能替代**支付平台官方收款码，除非内容就是平台给的收款链接。

## 动态码：金额常在屏上

收银系统、扫码枪、顾客屏上显示的订单码，常含订单号与金额，一笔一码或短时有效，防改金额。见 [链过期](https://gegegj.com/blog/qrcode-scans-but-link-expired)。

见 [美化收款码](https://gegegj.com/blog/qrcode-collection-code-style-still-scan)、[扫不上](https://gegegj.com/blog/qrcode-scan-fail-blur-size-light)。

## 店主该用哪种码

静态个人收款码长期有效，适合固定摊位；动态订单码一笔一码，适合线上订单防篡改金额。

把静态码截图发到网上，金额靠对方手动输入，有误付风险；动态码带订单号可对账。

聚合收款（多通道一码）由支付机构提供，与自己在 二维码生成器 做的普通 URL 码不是同一类。

见 链过期、收款码美化。

## 消费者防骗提示

付款前看清收款方名称是否与店铺一致，静态码被换贴的案例时有发生。

动态码若金额固定显示在屏上，再扫；手动改金额的静态码要核对数字。

短信里的「收款链接」可能是钓鱼，别与店内张贴码混信。

见 扫不上 技术因素排查。

自己开店测试：屏扫通再打印，见 打印留白。

## 使用提醒

本文只讨论标题里的一个具体场景，二维码生成器 在浏览器本地计算或处理，适合自查与沟通；签约、报税、诊疗、签证请以主管机构或合同为准。

涉及隐私的数据请先脱敏；公共电脑、录屏环境勿粘贴真实工资、病历、Token 或合同金额。

## 小结

店里贴墙的多为静态收款码；换海报不影响扫描，除非换错码图。大额收款核对对方昵称；动态码看屏上金额再付。

支付规则以平台说明为准。
