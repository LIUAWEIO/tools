# 换汇前先看牌价：买入、卖出和中间价，各自回答的是哪一句问话

**在网站上阅读：** [https://gegegj.com/blog/longread-exchange-rate-basics](https://gegegj.com/blog/longread-exchange-rate-basics)

**相关工具：** [打开工具页](https://gegegj.com/calc/exchange)

## 牌价表里，每一列其实在回答不同问法

汇率回答的是「一单位外币值多少本币」这类比价问题；我国日常语境里常见直接标价：100 美元折合若干人民币。

银行对客户会挂买入价（银行买入外币）和卖出价（银行卖出外币），中间夹着价差；新闻里的中间价多作参考锚，不等于您窗口成交的那个数。

即期、远期、现钞、现汇，对应不同交割与凭证形态；旅游、海淘、企业套保看到的「汇率」未必来自同一产品。

## App、机场屏、财经曲线：读数前先找纵轴单位

小数位从四位到六位不等；现钞与现汇不同价时，取错列会直接差一截。

走势图纵轴有时是「100 外币」有时是「1 外币」，截图发群里前先看一眼单位，避免传错情报。

## 做预算和做成交，是两段不同旅程

出境前粗算零花钱、海淘比价、把外币报价折成人民币心里底线——用公开牌价或手输假设汇率都够用。

若收入里含外币，可先粗换再和 [个税](https://gegegj.com/calc/tax)、[房贷](https://gegegj.com/calc/mortgage) 放在一张现金流表里看压力；真正换汇仍以柜台或网银成交价为准。

## 网页换算帮您少犯哪类错

「该乘还是该除」手一抖就反；工具把币种方向钉死，减少符号错误。临时假设「若贬到某价」也可手输汇率做沙盘。

鸽鸽工具网数据来自公开接口，与具体网点价差不同；大额或时效敏感场景，请以成交回执为准。

## 典型踩坑：用错一列价差出一截

用中间价锁合同、用卖出价去结汇，成本线会悄悄挪位；大额基数下少取一位小数也会堆出可见误差。

## 数据更实时之后，个人还要注意什么

API 与开放银行会让牌价刷新更快，但「参考价」和「成交价」仍会并存；养成「截图带时间戳」的习惯，对账更省事。
