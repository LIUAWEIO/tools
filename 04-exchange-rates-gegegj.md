# 汇率换算器｜鸽鸽工具网

**立即使用：** [https://gegegj.com/calc/exchange](https://gegegj.com/calc/exchange)

## 功能介绍

在人民币（CNY）基准下，将任意支持币种金额换算为目标币种；支持**刷新实时汇率**与**手动汇率**两种模式，兼顾「查牌价」与「公司内部固定报销汇率」两类需求。

## 支持的参数

| 参数 | 说明 |
|------|------|
| 金额 | 待换算的数值 |
| 源币种 / 目标币种 | 下拉选择；列表含 CNY 与接口返回的代码集合 |
| 实时汇率 | 依赖网络，点击「刷新汇率」拉取最新数据 |
| 手动汇率 | 勾选后为各外币维护「1 人民币 = X 单位外币」正数；代码格式为大写字母 2～10 位 |
| 交叉换算 | 非 CNY 币种间通过人民币基准间接换算 |

页面提示：数据来自公开接口，**仅供参考，成交请以银行为准**。

## 使用场景

- 出国旅行、海淘、跨境电商的成本粗算。
- 财务报销按公司规定汇率填表前的快速核对。
- 外贸报价阶段的数量级估算（正式合同需约定汇率源）。

## 如何使用

1. 打开 [汇率换算器](https://gegegj.com/calc/exchange)。
2. 等待加载完成或点击「刷新汇率」；失败时可改用手动汇率并填写各行。
3. 选择源币种、目标币种，输入金额，查看换算结果（通常保留四位小数展示）。
4. 重要付款前用银行或支付机构实际成交价复核。

## 常见问题

**页面汇率和银行成交价为什么不同？**  
银行区分现钞/现汇、买入卖出价，并可能含手续费；公开接口多为中间参考价，存在时滞。

**跨币种怎么算？**  
以 CNY 为枢纽：先将源币种折人民币，再折目标币种。若某币种缺汇率，需刷新或补充手动表。

**无网络能用吗？**  
可仅使用手动汇率模式（需自行维护各币种对人民币的换算系数）。

## 数据来源

- **服务端代理**（避免浏览器 CORS）：优先请求 `https://api.frankfurter.app/latest?from=CNY`，失败则尝试 `https://open.er-api.com/v6/latest/CNY`。
- 返回的 `rates` 表示 **1 人民币可兑换的外币数量**（与站内 `exchange.ts` 注释一致），响应 HTTP 缓存约 **300 秒**。
